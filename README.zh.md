# 性能监测猫猫 · Performance Cat

> For English, see [README.md](README.md).

一个轻量、原生的 **macOS 性能监测面板**，专为 Apple Silicon 打造。一个常驻窗口，
实时展示这台 Mac 当前的状态——CPU、功率、温度、风扇、内存、网络、存储和电池——
采用干净的、苹果风格的深色界面。

它用纯 **AppKit、单个 Swift 文件**实现。没有 Electron、没有网页视图、没有后台
守护进程、不联网、无统计上报，软件里也没有语言切换按钮。它只**读取**本地系统指标
并绘制出来。

> 资源占用：稳定后约 **1% CPU**、内存恒定在 **~70 MB**（活动监视器口径）；内置的
> `macmon` 传感器助手额外约 **1% CPU、~6 MB**。整个面板每次刷新先画进一张离屏位图，
> 再通过视图图层显示，所以内存是平的，不会每次重绘都飙升。

## 两个语言版本

软件内部不做语言切换——为了让程序保持精简，项目分别提供**两个独立的 app**，按需下载：

- **中文** —— `性能监测猫猫.app`
- **English** —— `Performance Cat.app`

两者除了界面语言外完全一致。

## 功能

每个数值都有清晰标注，一眼就知道是什么。

| 卡片 | 展示内容 |
|------|---------|
| **处理器** | 总负载 %、用户/系统/闲置占比、芯片型号、CPU 温度、实时折线 |
| **功率** | 系统功率（W）、SoC 合计，以及 CPU / GPU / ANE / DRAM 分项、温度状态 |
| **散热** | **从 SMC 直接读取的每个风扇真实转速**（风扇停转时如实显示 `0 RPM`），加上 CPU/GPU 温度；无风扇机型自动退化为温度面板 |
| **内存** | 已用 / 总量、压力 %、Swap、可用，**外加内存占用前三的应用** |
| **网络** | 实时上/下行吞吐，以及**流量占用前三的进程**（每秒） |
| **存储空间** | 仿苹果的启动卷分类占用：**系统 / 数据 / 其他 / 可用**，来自 `diskutil` |
| **电池** | 电量 %、充放功率、适配器功率、循环次数 |
| **AI 工具** | 对正在运行的 `codex` / `claude` 进程：**CPU %、内存、子进程数、已运行时长** |
| **活跃进程** | 当前 CPU 占用最高的进程 |

顶部状态条会显示一行综合状态（运行平稳 / CPU 高负载 / 散热压力升高 …）及关键指标
（SoC 功率、CPU/GPU 温度、内存 %）。

## 数据从哪来

性能监测猫猫对每个数字的来源都很诚实，绝不编造：

- **CPU / 内存 / 网络吞吐 / 电池** —— 苹果系统 API
  （`host_processor_info`、`host_statistics64`、`getifaddrs`、IOKit 电源）。
- **风扇转速** —— 通过 IOKit 直接读 **SMC**（`AppleSMC`），只读、无需特权。机器凉的
  时候风扇本就停转，所以会如实显示 `0 RPM`。
- **功率与温度** —— 内置的 [`macmon`](https://github.com/vladkens/macmon) 助手，
  作为子进程启动并解析其 JSON 输出。
- **进程级网络** —— `nettop`，在后台线程上短暂采样。
- **存储分类** —— `diskutil apfs list`，约每 5 分钟采样一次。

## 环境要求

- macOS 14 (Sonoma) 或更高，Apple Silicon。
- 从源码构建需要 **Xcode 命令行工具**：`xcode-select --install`。

## 构建与运行

```bash
git clone <你的仓库地址> performance-cat
cd performance-cat
./build.sh                 # 把中英两个版本都构建到 dist/
# 也可以只构建一个：  ./build.sh zh   |   ./build.sh en

codesign --force --deep --sign - "dist/性能监测猫猫.app"
open "dist/性能监测猫猫.app"
```

`build.sh` 会编译那个单文件 Swift 源（按语言各编译一次）、生成 app 图标，并把 `macmon`
助手复制进每个 app 包里。产物是一个自包含的 `.app`，可以拖进 `/Applications` 一直开着。

临时的 `codesign` 让 app 无需付费开发者证书就能本地运行。首次打开时 macOS 可能提示
"来自身份不明的开发者"——右键 → 打开 即可。

## 权限与隐私

- 本应用**只读取**系统指标，从不修改任何系统设置，也**从不把任何数据发往任何地方**——
  里面根本没有联网代码。
- 不需要任何特殊授权。macOS 可能会就普通访问（如启动助手工具）弹窗，想要传感器读数就同意。
- 内存 / 网络 / 活跃进程 / AI 卡片里的进程和应用名来自 `ps` / `nettop`。不读取、不存储
  任何文件内容、对话内容或个人数据。
